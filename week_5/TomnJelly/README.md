### README 스터디 내용 정리 세부 주제

- Class 정의하는 방법
  자바에서 클래스는 접근 제어자와 함께 class 키워드를 사용하여 선언합니다.
  class 클래스명{} 와 같은 형태로 사용
  내부에는 필드, 생성자, 메소드를 정의 가능

- 객체 만드는 방법 (keyword new에 대한 이해)
  Tv t; // 클래스 타입의 참조변수 t를 선언
  t = new Tv(); // Tv 인스턴스를 생성한 후, 생선된 Tv 인스턴스의 주소를 t에 저장

  new 연산자
  : 인스턴스(객체)를 생성해주는 역할; 메모리에 데이터를 저장할 공간을 할당받고 공간의 참조값을 객체에게 반환하여 주고 이어서 생성자를 호출

  사용법
  : 클래스 변수 = new 클래스();

- method 정의하는 방법
  메서드 : 특정 작업을 수행하는 일련의 문장들을 하나로 묶은 것

  반환타입 메서드 이름(타입 변수명, 타입변수명, ...)
  {
  // 메서드 호출시 수행될 코드 //
  }

  인자 : 메서드를 호출할 때 괄호 안에 지정해준 값
  매개변수 : 메서드를 선언할 때 정의한 것

- constructor 정의하는 방법
  생성자(constructor)는 객체가 생성될 때 자동으로 호출되는 특수 목적의 메소드로 '객체의 초기화'를 위해 사용합니다.
  생성자의 이름은 반드시 클래스의 이름과 같아야 하고 리턴타입이 없으며, void 키워드를 사용하지 않는다.

- this 키워드 이해하기
  JAVA에서 this는 객채, 자기자신을 나타내고 사용하는 상황은 크게 3가지
  1. 클래스의 속성과 생성자/메소드의 매개변수의 이름이 같은 경우
     : 보통 생성자나 setter,getter 메서드의 매개변수 이름을 클래스의 속성 이름과 동일하게 적용되기 때문입니다.
  2. 클래스의 오버로딩된 다른 생성자 호출
     : 햐냐의 클래스에 여러개의 생성자가 오버로딩되어 있을 때 서로 중복된 코드를 가지고 있는 경우가 꽤 있기 때문에 내부에 정의된 다른 생성자를 호출하여 중복을 피하는 것입니다.
  3. 객체 자신의 참조값을 전달하고 싶을 때
     : 어떤 메소드에서 동작을 완료하고 리턴값으로 메소드의 매개변수로 객체, 자기 자신의 참조값을 전달하고 싶어하는 경우에 사용합니다.
     class Constructor{
     //생성자
     Constructor(매개변수){
     ...
     }
     }
